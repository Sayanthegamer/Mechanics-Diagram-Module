# Phase 9 Plan 2: Interaction & Parameter Bindings

## Objective
Implement hit-testing and dragging interaction handlers on the schematic canvas for switches and component selections, and dynamically hook selected component attributes to sidebar controls.

## Execution Context
- **Tech Stack:** Vanilla TypeScript + Vite + CSS (no external styling or design systems).
- **Core Guidelines:** Honor exact spacing tokens, fonts, and colors from `09-UI-SPEC.md`.

## Context
- `src/main.ts` manages core user mouse interaction handlers (`mousedown`, `mousemove`, `mouseup`) and dynamic sidebar slider renders.
- `src/lib/diagrams/circuit/circuit.ts` manages spatial node indexes and matrix re-evaluations.

## Tasks
1. **Type**: implementation
   **Name**: Implement canvas interaction for switches and components
   **Files**:
     - `src/main.ts`
   **Action**: Extend canvas click/tap listener to perform hit testing on schematic components. If a switch is clicked, toggle its `closed` state, then run `circuitEngine.analyzeCircuit()` to refresh transient node matrices (D-04). If a parametric component (resistor, capacitor, inductor, voltage source) is clicked, mark it as the selected target (D-05).
   **Verify**: Clicking a switch toggles its schematic open/closed position and updates node potential colors.
   **Done**: Interaction triggers topological MNA updates on switch click.

2. **Type**: implementation
   **Name**: Render selection highlight on canvas
   **Files**:
     - `src/lib/diagrams/CircuitDiagram.ts`
     - `src/main.ts`
   **Action**: Draw a glowing outline or dash ring around the selected component using accent color `#6366f1` (indigo) (D-05). Add outline/border glow mapping to render loop when drawing the active element.
   **Verify**: Clicking a resistor or capacitor renders a visual bounding highlight around it.
   **Done**: Active selection shows high-contrast accent color highlighting.

3. **Type**: implementation
   **Name**: Bind component attributes dynamically to sidebar sliders
   **Files**:
     - `src/main.ts`
   **Action**: Hook the selected component's internal parameters (e.g. resistance, capacitance, inductance, voltage amplitude) into `renderSliders` in `src/main.ts` (D-05). Any slider adjustment must dynamically update the active element parameter in the circuit data structure and trigger real-time solver steps.
   **Verify**: Dragging the resistance slider dynamically alters matrix elements in the telemetry inspector.
   **Done**: Real-time param modification synchronizes sliders and canvas simulation parameters.

## Verification
- Verify click events on switch toggle topologically recalculate solver paths.
- Verify slider adjustments on selected components redraw and recalculate circuit transient matrices on the fly.
