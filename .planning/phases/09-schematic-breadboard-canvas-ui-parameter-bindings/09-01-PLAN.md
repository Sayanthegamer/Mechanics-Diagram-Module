# Phase 9 Plan 1: Schematic Breadboard Canvas Rendering & Animations

## Objective
Implement schematic component drawing routines directly inside `src/lib/diagrams/CircuitDiagram.ts` along with dynamic potential-based coloring and animated current dots.

## Execution Context
- **Tech Stack:** Vanilla TypeScript + Vite + CSS (no external styling or design systems).
- **Core Guidelines:** Honor exact spacing tokens, fonts, and colors from `09-UI-SPEC.md`. Maintain the exact dynamic potential-based colors and animations specified in `09-CONTEXT.md`.

## Context
- `scratch/circuitjs/circuit-sim/src/renderer/element-renderers.ts` provides canonical component line drawing geometry.
- `scratch/circuitjs/circuit-sim/src/renderer/voltage-colors.ts` shows voltage-to-color mapping parameters.
- `scratch/circuitjs/circuit-sim/src/renderer/current-dots.ts` outlines charge particle animation formulas.

## Tasks
1. **Type**: implementation
   **Name**: Port and house schematic drawings in CircuitDiagram.ts
   **Files**:
     - `src/lib/diagrams/CircuitDiagram.ts`
   **Action**: Create `src/lib/diagrams/CircuitDiagram.ts` containing the `CircuitDiagram` class. Implement standard schematic drawing routines for `resistor`, `switch` (open/closed), `capacitor`, `inductor`, `voltage` source, `ground`, and `wire` using Canvas2D (D-01).
   **Verify**: Ensure all geometry drawing matches the source rendering vectors in `element-renderers.ts`.
   **Done**: File compiled without TypeScript errors.

2. **Type**: implementation
   **Name**: Implement potential-based voltage coloring
   **Files**:
     - `src/lib/diagrams/CircuitDiagram.ts`
   **Action**: Implement voltage-to-color dynamic mapping (D-02) matching colors in `09-UI-SPEC.md`:
     - Positive Potential (V > 0V): `#ef4444` (bright red)
     - Neutral Potential (V = 0V): `#505050` (neutral green/gray)
     - Negative Potential (V < 0V): `#3b82f6` (bright blue)
   **Verify**: Ensure continuous gradient scaling for potentials using `voltageToColor` mapped values.
   **Done**: Wire/lead segments render with voltage potential color gradients.

3. **Type**: implementation
   **Name**: Implement current dot particle animations
   **Files**:
     - `src/lib/diagrams/CircuitDiagram.ts`
   **Action**: Add `drawCurrentDots` to visual renderers (D-03) using golden amber color `#fbbf24` for moving dots. Calculate particle offset and direction based on computed node currents and simulation elapsed time.
   **Verify**: When simulation runs, check that charge particles animate on active loops and stay stationary when current is zero.
   **Done**: Animated current dots move down lines matching the exact speed formula from `current-dots.ts`.
