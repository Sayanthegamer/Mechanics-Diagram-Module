# 06-01-PLAN: Core Integration & Charge Interaction UI

This plan establishes type safety definitions, integrates `EmDiagram` within the core lifecycle, and implements interactive user controls (adding, selecting, dragging, adjusting, and deleting charges).

<objective>
Configure the electrostatics types and integrate EmDiagram into the main application. Implement mouse/touch drag-and-drop interaction to select, move, and configure point charges on the canvas, showing telemetry and custom controls in the sidebar panel.
</objective>

<execution_context>
The project uses Vanilla TypeScript, Vite, and HTML5 Canvas. Code styling follows semicolons and 2-space indentation.
</execution_context>

<context>
The sandbox relies on EmDiagram.ts which currently holds placeholder code. To support point charges, we need to register the `'em'` diagram type, update preset choices, and capture pointer actions (mouse/touch events) mapped to the canvas.
</context>

<threat_model>
- **Null Reference on Drag**: Selecting or editing coordinates of a deleted charge must be guarded to prevent runtime crash.
- **Out of Bounds Charges**: Limit drag boundaries or handle charge relocation gracefully so charges cannot be dragged infinitely off-screen.
</threat_model>

<verification>
