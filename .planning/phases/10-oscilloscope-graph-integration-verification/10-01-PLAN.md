# Phase 10 Plan 1: Telemetry Extension and Oscilloscope Rendering

## Objective
Extend the circuit solver telemetry and implement the core dual-channel oscilloscope plotting modes (time-series Y-T and phase-space X-Y) inside the graph rendering module.

## Execution Context
- **Tech Stack:** Vanilla TypeScript + Vite + CSS (no external styling or design systems).
- **Core Guidelines:** Zero external UI libraries. Use HTML5 Canvas2D rendering for precise telemetry curves. Honor standard dynamic auto-scaling rules and color configurations.

## Context
- `src/lib/diagrams/circuit/types.ts` defines telemetry structures (`ElementState` interface).
- `src/lib/diagrams/circuit/circuit.ts` manages the simulation state step solving and telemetry generation (`getState()`).
- `src/lib/diagrams/GraphModule.ts` houses the plotting engine and holds the grid/axes geometry drawing methods.

   **Files**:

   **Verify**: Ensure the project compiles without TypeScript errors.

## Threat Model

### Threats & Mitigations
- **Division by Zero (NaN Canvas Coordinates)**:
  - *Threat*: If a circuit element has 0V differential voltage, the dynamic scale range calculation `yMax - yMin` evaluates to 0. Dividing by this range inside coordinate transformations yields `NaN` or `Infinity`, causing browser canvas errors or empty charts.
  - *Mitigation*: Clamp the dynamic scaling range. If `yMax - yMin < 0.01`, programmatically override the boundaries to `yMin -= 0.5; yMax += 0.5;` to enforce a minimum scale height of 1.0.
- **Null Element Telemetry Dereference**:
  - *Threat*: If a selected component is deleted or replaced, passing its ID to the telemetry reader could result in looking up undefined states, throwing runtime exceptions.
  - *Mitigation*: Wrap lookups with strict fallback checks. If the selected element cannot be found in the active telemetry step, return default values of 0.
