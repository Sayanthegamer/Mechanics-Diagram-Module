# Phase 10 Plan 2: Orchestration, Selection Overrides, and UI Dropdown Bindings

## Objective
Update the application orchestrator to store detailed component state history, connect interactive selection overrides, enable dropdown selectors in the UI, and log validation checklists.

## Execution Context
- **Tech Stack:** Vanilla TypeScript + Vite + CSS (no external styling or design systems).
- **Core Guidelines:** Wire selections must toggle default views dynamically. Clean up old elements and refresh simulation buffers when changing presets.

## Context
- `src/main.ts` manages the main loop, active configs, preset configurations, and draws the primary simulation canvas.
- `index.html` houses the sidebar panels and top toolbar controls.
- `.planning/phases/10-oscilloscope-graph-integration-verification/10-VALIDATION.md` dictates the manual verification criteria and feedback frequencies.

## Tasks

   **Name**: Capture element states history and selection overrides

## Threat Model

### Threats & Mitigations
- **Null Selector Reference Crash**:
  - *Threat*: If elements are queried from `index.html` but fail to resolve (due to structural modifications), attaching event listeners will throw fatal null reference errors.
  - *Mitigation*: Implement strict null checks when fetching `selectGraphMode` or other DOM elements in `src/main.ts` to ensure the program fails gracefully.
- **Memory Pressure from Rich Telemetry Buffering**:
  - *Threat*: Storing rich telemetry objects instead of scalar numbers inside `circuitHistory` could lead to high memory consumption if the queue size is not capped.
  - *Mitigation*: Enforce a strict buffer ceiling of 500 elements. Old items are aggressively evicted via `.shift()` to maintain a flat memory graph.

## Verification
- Build and compile the workspace with `npm run build` or Vite compiler to verify no syntax or type errors.
- Visual validation of selection override: verify that clicking components swaps graph lines, and clicking empty space restores defaults.
- Visual validation of dropdown controls: verify that changing the dropdown switches between YT time-series plot and XY Lissajous ellipse.

## Success Criteria
- Telemetry buffer saves the state history of all components.
- Selection overrides plot Channel A & B for selected component.
- Dropdown options exist and are operational, switching chart rendering.
- Visual verification protocols are verified.
