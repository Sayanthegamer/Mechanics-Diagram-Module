# Plan 1: Extend EmConfig & Implement Interactive Particle Gun Turret UI

## <objective>
Set up the typescript interfaces and configuration properties for Phase 7 (Lorentz Force & Magnetic Deflections) and implement the interactive particle gun UI. This includes rendering the gun on the canvas, enabling coordinate translation to aim and reposition it via mouse/touch drag events, and integrating manual sidebar sliders/buttons to control particle firing.
</objective>

## <execution_context>
The project tech stack uses Vanilla TypeScript + Vite + Vanilla CSS. Spacing scale, typography, and colors are defined in `07-UI-SPEC.md`. Accent colors like `#a855f7` are designated for the particle gun turret. Canvas2D API is used directly. No external libraries are used for UI controls.
</execution_context>

## <context>
- `src/lib/types.ts` defines `EmConfig` and related types.
- `src/lib/diagrams/EmDiagram.ts` implements the electrostatics simulation state and drawing loop.
- `src/main.ts` orchestrates controls, canvas mouse events, and configuration bindings.
- Decisions from `07-CONTEXT.md` require hybrid controls: both a draggable gun on canvas (D-01) and manual sidebar controls (D-02).
</context>

## <threat_model>
- **Boundary Validation**: Dragging the particle gun could place it off-screen, making it inaccessible.
  - *Mitigation*: Clamp `gunX` and `gunY` coordinates to the canvas bounds (e.g. $x \in [-8, 8]$, $y \in [-6, 6]$).
- **Infinite Particle Firing**: User fires unlimited particles causing memory exhaustion.
  - *Mitigation*: Limit the maximum active particle count to 20. If exceeded, remove the oldest particle.
</threat_model>
