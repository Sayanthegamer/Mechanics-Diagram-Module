# Plan 8-2: Circuit Solver Engine Integration & Test Setup

## <objective>
Port the main `Circuit` solver class, hook up the time-stepping simulation loop to the animation frame routine in `src/main.ts`, and run validation checks with static preset circuits to ensure dynamic solving capabilities.

## <execution_context>
The project uses Vanilla TypeScript + Vite. The main loop inside `src/main.ts` runs physics simulations at 60fps using delta time (`dt`) updates.

## <context>
- Requires files ported in Plan 1 (`matrix.ts`, `types.ts`, `elements/`).
- The `Circuit` class acts as the orchestrator: parsing circuit graphs, sorting nodes, managing switch states, invoking element stamps, and calling the LU decomposition solver.
- The solver output must be fed back to telemetry or state variables without rendering visual diagrams (rendering is deferred to Phase 9).

## <threat_model>
- **Infinite Solving Loops (Simulation Overload / DoS)**: Non-linear components or floating nodes might trigger infinite Newton-Raphson iterations, freezing the browser animation frame.
  - *Mitigation*: Limit the maximum number of Newton-Raphson solver iterations per step (e.g. standard limit of 100) and step count limits per animation frame to prevent page hangs.
- **Varying dt Instabilities**: Extreme physics step differences (`dt`) can cause integration errors or matrix singularity.
  - *Mitigation*: Cap the maximum simulation timestep duration and sub-step the math engine in fixed-size intervals (e.g. `5e-6` seconds).
</threat_model>

## <verification>

- **Manual**: Load the application with `npm run dev` and select the static circuit preset to confirm numerical values resolve correctly.
</verification>

- Preset configurations calculate values (currents, voltages) for resistor, capacitor, and inductor networks in real-time.
- Newton-Raphson iterations converge cleanly without page freezing.
