## Concerns
- **Numerical Integration drift**: Standard Euler integration works well for high frame rate client-side visuals. However, if extreme parameters (high density, heavy block mass, minimal damping) are entered, some numerical oscillations might drift over long periods.
- **Atmospheric Pressure Probe**: Atmospheric pressure (101.3 kPa) is large relative to depth-based gauge pressure (e.g. 10–20 kPa at standard depths). This scales graph curves significantly. However, since the toggle switch hides/shows it and the Y-axis auto-scales, this has been mitigated successfully.

## Recommendations
1. Keep the standard Euler solvers as the default since they are lightweight and run smoothly at 60fps on typical client hardware.
2. Consider adding an RK4 integration option in future refactoring for advanced physics classes requiring exact numerical convergence.
