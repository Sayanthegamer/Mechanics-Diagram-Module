## Lessons Learned
- **Visual vs Physical Scaling**: In hydrodynamics (specifically pipe flow and falling sphere viscosity), raw physical speed is often very small (e.g. < 0.02 m/s), which renders at < 1 pixel/second. Applying a clean visual multiplier (e.g. 15x) for particle advection while keeping the underlying physics equations intact is key to creating engaging interactive visualizations.
- **Unified Draw Method Refactoring**: By passing the active mode string to `drawFluids(history, mode)`, we kept `GraphModule.ts` clean and decoupled from the orchestrator `main.ts` configurations.
- **Centralized Viewport Transform**: Viewport panning was implemented globally in `PhysicsCanvas.ts`'s coordinate mapping methods (`toScreen`/`toPhysics`), which automatically added support to all existing mechanical and wave presets without modifying their individual draw methods.
