## Overview

The Physics Diagrams & Simulation Module is a framework-free interactive web application designed for high-performance visual simulations of classical mechanics, simple harmonic motion, wave mechanics, and vector operations. Built using vanilla TypeScript and Vite, it utilizes the HTML5 Canvas 2D API to render dynamic, visually polished, and textbook-accurate diagrams.

## Components

### 1. Orchestrator
- **File**: `src/main.ts`
- **Purpose**: Initializes the UI dashboard, binds event listeners for parameters and presets, maintains the main simulation state, and runs the `requestAnimationFrame` loop. It applies JSON configurations, updates the sidebar parameters, and syncs rendering across the main simulation canvas and graph canvas.

### 2. Core Drawing Engine
- **File**: `src/lib/PhysicsCanvas.ts`
- **Purpose**: A utility wrapper around the HTML5 Canvas 2D context. It manages physics-to-screen coordinate transforms, viewport panning offsets (`panX`, `panY`), grid layouts (both cartesian and polar), vector arrow rendering (with components, labels, and arcs), and complex geometric primitives (blocks, pulleys, coils/springs, and inclined wedges).

### 3. Diagram & Simulation Engines (`src/lib/diagrams/`)
Each diagram class follows a unified interface: `setConfig(config)`, `resetState()`, `step(dt)`, and `draw(canvas)`.

- **FbdDiagram.ts**:
  - Simulates blocks on flat, inclined, or suspended surfaces under Euler integration.
  - Draws dynamic force vectors (Gravity, Normal, Friction, Tension, Applied force) acting on the center of mass.
  - Supports gravity component decomposition on inclined planes ($mg\sin\theta, mg\cos\theta$).
- **VectorDiagram.ts**:
  - Displays 2D and 3D vectors.
  - Visualizes algebraic operations (vector addition, subtraction, dot product projection, and cross product in a pseudo-3D isometric space).
- **ShmDiagram.ts**:
  - Simulates spring-mass systems (horizontal and vertical) and pendulums.
  - Implements numerical integration (Euler-Cromer) with support for linear damping ($b \cdot v$) and external driving forces ($F_d \cos(\omega t)$) to demonstrate resonance.
- **WaveDiagram.ts**:
  - Renders wave patterns including transverse sinusoidal waves, longitudinal compression waves (using line density), wave packet/pulse superposition, and standing waves with labeled nodes (N) and antinodes (A).
- **MechanicsDiagram.ts**:
  - Simulates macroscopic classical mechanics systems:
    1. Projectile motion with quadratic air drag ($b \cdot v^2$).
    2. Pulley systems (Atwood machines and inclined planes) with mass-tension acceleration physics.
    3. Collisions (1D and 2D) using coefficient of restitution ($e$) and particle spark effects on impact.
    4. Circular motion (vertical loops with conservation of energy tension $T(\theta) = \frac{mv^2}{R} + mg\cos\theta$ and uniform horizontal orbits).
- **FluidsDiagram.ts**:
  - Simulates fluid mechanics scenarios (Buoyancy floating block lab with pressure probe, Pascal hydraulic press, Bernoulli pipe Venturi flow, and Viscous drag falling sphere).
  - Calculates submerged block volumes, buoyancy force vectors, and viscous fluid damping to stabilize oscillations.
  - Implements hydraulic cylinder volume conservation constraint equations.
  - Solves Bernoulli Venturi velocity and pressure columns.
  - Models falling sphere force vectors (gravity, buoyancy, drag) and terminal velocity convergence.
- **GravityDiagram.ts**:
  - Simulates dynamic gravitational systems including Keplerian orbit sector sweeping, Two-Body barycentric orbits, and Escape velocity probe launches.
  - Implements Kepler's solver (Newton-Raphson) and high-accuracy RK4 ODE integrator.
  - Calculates kinetic and potential energy components in real-time.
- **ThermoDiagram.ts**:
  - Simulates thermodynamic and kinetic theory systems with three modes: Kinetic Theory of Gases (Rayleigh speed distributions, gas mixtures), Piston Engine (Carnot, isothermal, isobaric, isochoric, and adiabatic processes), and Gas Diffusion (barrier removal, particle mixing).
  - Models particle-wall collisions with impulse tracking to calculate physical pressure, and piston dynamics with volume conservation.
  - Calculates entropy ($S = 1.5 \ln T + \ln V$) for T-S diagrams.
- **GraphModule.ts**:
  - Plots real-time data points (kinematics, energies, phase space orbits, fluid displacements, flow speeds, pressures, and thermodynamic cycles) on a separate HTML5 Canvas.
  - Supports customized scaling, legends, and specialized graphing modes including real-time P-V / T-S trace overlays and dynamic Maxwell-Boltzmann speed distributions with theoretical Rayleigh curves.

### 4. Interactive & Accessibility Subsystem

## Data Flow

1. **Preset Selection**: The user selects a preset in the dropdown (or edits the raw JSON in the editor).
2. **Configuration Load**: `main.ts` loads the configuration, generates matching UI sliders dynamically, and passes the config to the active diagram module via `setConfig(config)`.
3. **Simulation Loop**:
   - `stepSimulation()` advances the active diagram's physics state by `dt` via its `step(dt)` method.
   - `drawActiveSimulation()` clears the canvas and invokes the diagram's `draw(canvas)` method.
   - The diagram leverages `PhysicsCanvas` API methods to draw vectors, blocks, springs, etc.
   - If enabled, state data is fed to `GraphModule` to render real-time plots, and state statistics are written to the bottom status bar.

## Technical Debt & Conventions

- **State Management**: Animation states and histories are stored locally within each diagram instance and main runner. There is no external state library.
- **Physics Solver**: Numerical integrations use standard Euler/Euler-Cromer. While sufficient for real-time visualization at high frame rates, they can accumulate error over long runs under extreme force/damping conditions.
- **Conventions**:
  - Standard SI Units used throughout: meters (m), kilograms (kg), seconds (s), Newtons (N).
  - Canvas coordinate system: Custom cartesian coordinate system (Y-up, origin centered or bottom-centered depending on preset) handled by `PhysicsCanvas.toScreen()`.
